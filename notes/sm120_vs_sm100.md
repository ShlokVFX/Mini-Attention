# SM120 vs SM100: Consumer vs Datacenter Blackwell

**One sentence:** SM120 (RTX 5090) does not expose `tcgen05.mma` — Tensor Memory (TMEM) and WGMMA require datacenter SM100 hardware.

---

## Hardware mapping

| Feature                    | SM100 (B200 / GB200)         | SM120 (RTX 5090 / GB205)     |
|----------------------------|------------------------------|------------------------------|
| Architecture               | Blackwell datacenter         | Blackwell consumer           |
| tcgen05 / TMEM             | ✓ 512 KB/SM                 | ✗ not exposed                |
| WGMMA (warp-group MMA)     | ✓                            | ✗                            |
| mma.sync.aligned           | ✓                            | ✓ (sole tensor-core ISA)     |
| FP4 tensor cores           | ✓ (MXFP4, NF4)              | ✓ (MXFP4, NF4)              |
| L2 cache                   | ~80 MB                       | 96 MB                        |
| Memory bandwidth           | 8 TB/s (HBM3e)               | 1.8 TB/s (GDDR7)             |
| FP16 peak                  | ~1800 TFLOP/s                | 419 TFLOP/s                  |

---

## What tcgen05 / TMEM means

`tcgen05.mma` is the PTX instruction that feeds the **Tensor Memory Accumulator (TMEM)** pipeline on SM100. TMEM is a 512 KB/SM scratchpad register file separate from shared memory. It enables persistent warp-specialization: producer warps issue async MMA descriptors, consumer warps read TMEM results without explicit synchronisation. FlashAttention-3's warp-specialized pipeline depends on this path.

None of this is accessible on SM120.

---

## What SM120 actually uses

On SM120 the only tensor-core dispatch path is `mma.sync.aligned.m16n8k16` — the same Ampere-vintage instruction available on SM86. SM120's advantage over SM86 is:

- 8× higher FP16 throughput (419 vs 51 TFLOP/s peak)
- 32× larger L2 (96 MB vs 3 MB)
- GDDR7 vs GDDR6 memory bandwidth

The SM120-specific tuning knobs used in `kernels/flash_attn/cuda/sm_120/`:

1. `L2::256B` cache hints — exploit the 96 MB L2 with wider sectors
2. `__rcp_approx_ftz_f32` — fast reciprocal unit, new on GB20x silicon
3. `Bc=128` outer tile — SM120 has 2× the shared memory per block vs SM86

---

## Common mistake to avoid

Several GitHub repos and papers conflate SM100 and SM120. If you compile with `-arch=sm_100` your kernel will **not run** on an RTX 5090. You must use `-arch=sm_120` (or the alias `sm_120a`). `nvcc` will silently accept `sm_100` and produce a CUDA error at device dispatch time.
